# Обоснование выбора финальной модели

**Выбрана: Random Forest**


## Критерий выбора

Основной критерий — **macro F1** (официальный критерий приёмки по заданию, порог >= 0.70). Recall для класса «3+» (студенты высокого риска) отслеживается отдельно как важный, но вторичный показатель: модель, максимизирующая только recall «3+», рискует слишком часто ложно относить благополучных студентов к группе риска (низкий precision), что на практике означало бы лишнюю нагрузку на кураторов.


## Сравнение всех вариантов

| Модель | Accuracy | Macro F1 | Weighted F1 | Recall «3+» |
|---|---|---|---|---|
| Logistic Regression (baseline) | 0.658 | 0.421 | 0.707 | 0.664 |
| Random Forest 🏆 | 0.741 | 0.482 | 0.770 | 0.701 |
| XGBoost (class_weight) | 0.704 | 0.466 | 0.748 | 0.674 |
| XGBoost (SMOTE) | 0.734 | 0.476 | 0.764 | 0.674 |
| CatBoost (class_weight) | 0.695 | 0.465 | 0.738 | 0.711 |
| CatBoost (SMOTE) | 0.708 | 0.463 | 0.748 | 0.661 |

## Итог

**Random Forest** показала лучший macro F1 (0.482) среди всех 6 вариантов, при recall «3+» = 0.701.


**Важная оговорка:** критерий приёмки по заданию — macro F1 >= 0.70 — этой моделью пока НЕ достигнут (0.482 < 0.70). Модель сохранена как рабочий baseline для дальнейшей доработки (признаки, объединение классов «1»/«2», больше данных), а не как финальный production-результат.


**Альтернатива для обсуждения с заказчиком:** CatBoost (class_weight) даёт заметно более высокий recall «3+» (0.711 против 0.701 у выбранной модели), ценой немного более низкого macro F1 (0.465). Если для деканата принципиально важнее не пропустить студента высокого риска, чем избежать ложных срабатываний — стоит рассмотреть эту модель вместо текущего выбора (файл сохранён отдельно в models/).


## Как использовать

```python
import joblib
model = joblib.load('models/model.pkl')
predictions = model.predict(X)          # категория 0/1/2/3
probabilities = model.predict_proba(X)   # вероятности по каждому классу
```

Порядок классов: ['0', '1', '2', '3+'] (индексы 0-3 соответственно). Полный список и порядок признаков — см. reports/feature_contract.md.
