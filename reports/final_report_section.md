# Раздел отчёта: используемые алгоритмы, архитектура, метрики качества

## 1. Архитектура решения

Система построена как последовательный пайплайн из независимых, переиспользуемых модулей — каждый решает одну задачу и может запускаться отдельно (что удобно и для разработки, и для интеграции с backend):

```
data.xlsx (сырые данные об успеваемости)
        │
        ▼
src/features.py    → строит признаки "студент x семестр" (21 признак),
        │             считает и сохраняет карты сложности групп/дисциплин
        │             (models/difficulty_maps.pkl) для согласованного
        │             инференса на новых файлах
        ▼
src/dataset.py     → делит на dataset_train.csv (есть таргет) и
        │             dataset_predict.csv (последний семестр студента,
        │             таргет неизвестен — реальная прогнозная выборка)
        ▼
src/train.py        → обучает 4 варианта (XGBoost/CatBoost x
src/train_baseline.py  class_weight/SMOTE), Logistic Regression и
src/train_rf.py        Random Forest, все — с разбиением train/test по
        │               времени и TimeSeriesSplit-подбором гиперпараметров
        ▼
src/select_model.py → сравнивает все 6 моделей на одном holdout-тесте,
        │              сохраняет лучшую как models/model.pkl с обоснованием
        ▼
src/feature_importance.py       → экспорт важности признаков для отчёта
src/interpret_discrepancies.py  → качественный разбор ошибок модели
        ▼
src/predict.py     → применение model.pkl к новому загруженному файлу,
                      экспорт ранжированного списка студентов в Excel
                      (это и есть интерфейс для backend/веб-приложения)
```

Ключевое архитектурное решение — **разделение обучения и инференса через сохранённые артефакты** (`model.pkl`, `difficulty_maps.pkl`). Это было осознанно исправлено после того, как тестирование на новом (изолированном) файле показало: если пересчитывать статистику сложности групп/дисциплин заново на каждом новом файле, признаки становятся несовместимы с тем, на чём обучалась модель.

## 2. Используемые алгоритмы

Согласно заданию, обучены и сравнены **4 типа моделей**:

| Модель | Назначение по заданию | Балансировка классов |
|---|---|---|
| Logistic Regression | Бейзлайн для сравнения | `class_weight="balanced"` |
| Random Forest | Интерпретируемость | `class_weight="balanced"` |
| XGBoost | Максимальная точность | class_weight (sample_weight) **и** SMOTE — сравнены оба варианта |
| CatBoost | Максимальная точность | class_weight (`auto_class_weights`) **и** SMOTE — сравнены оба варианта |

Итого — **6 обученных и честно сравненных вариантов** на одном и том же holdout-тесте.

**Общие методологические решения, применённые ко всем моделям:**
- Разбиение train/test — **строго по времени** (ранние семестры → train, поздние → test, пропорция ~80/20), а не случайно — иначе была бы утечка данных из будущего.
- Подбор гиперпараметров XGBoost/CatBoost — через **TimeSeriesSplit-кросс-валидацию** (5 расширяющихся фолдов) внутри обучающей части, чтобы не «заглядывать» в будущее уже на этапе тюнинга.
- Целевая переменная — количество задолженностей в **следующем** семестре относительно текущего (сдвиг по времени), с исключением случаев, когда следующий семестр студента ещё не завершён (менее 50% выставленных оценок) — иначе такие случаи ложно выглядели бы как «0 задолженностей».

## 3. Метрики качества

**Критерий приёмки по заданию: macro F1-score ≥ 0.70.**

| Модель | Accuracy | Macro F1 | Weighted F1 | Recall «3+» |
|---|---|---|---|---|
| Logistic Regression (baseline) | 0.658 | 0.421 | 0.707 | 0.664 |
| **Random Forest** 🏆 | **0.741** | **0.482** | **0.770** | 0.701 |
| XGBoost (class_weight) | 0.703 | 0.466 | 0.747 | 0.668 |
| XGBoost (SMOTE) | 0.735 | 0.471 | 0.764 | 0.664 |
| CatBoost (class_weight) | 0.695 | 0.465 | 0.738 | **0.711** |
| CatBoost (SMOTE) | 0.709 | 0.464 | 0.749 | 0.658 |

**Выбрана модель: Random Forest** (лучший macro F1 среди всех вариантов). Полное обоснование выбора — в `reports/model_selection.md`.

**Честная оговорка:** порог приёмки (macro F1 ≥ 0.70) **пока не достигнут** (0.482 < 0.70). Текущий результат — рабочий baseline, требующий дальнейшей доработки признаков и/или данных (см. раздел «Рекомендации по улучшению»).

### Важность признаков (Random Forest)

Топ-5 наиболее значимых признаков: `group_difficulty` (сложность учебной группы), `avg_discipline_difficulty` (сложность пройденных дисциплин), `avg_score` (средний балл), `n_debts` (текущие задолженности), `prev_avg_score` (балл за прошлый семестр). Полная таблица и график — `reports/feature_importance.csv/.png`.

### Качественный анализ ошибок

- Из **топ-10** студентов с наибольшим прогнозируемым риском — **10 из 10** прогнозов подтвердились фактом.
- **29.9%** реальных случаев «3+» пропущены моделью — преимущественно это студенты, у которых текущих долгов почти не было (в среднем 0.39 против 3.99 у верно распознанных), т.е. модель хуже предсказывает **резкое** ухудшение успеваемости с благополучного старта.
- Ложных тревог (факт «0», прогноз «3+») — всего **4.5%**.

Полный разбор — `reports/discrepancy_analysis.md`.
